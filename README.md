# anonymyzr

A local privacy proxy for the Anthropic Claude API.

Sits between your AI client (OpenClaw, custom apps, etc.) and Anthropic's servers. Detects and replaces PII in outgoing requests using a **local** NER model â€” your real data never leaves your machine. Restores original values in the response before returning it to you.

## How it works

1. **Detect** â€” [GLiNER](https://github.com/urchade/GLiNER) scans outgoing messages for PII (names, addresses, SSNs, phone numbers, dollar amounts, etc.) entirely on-device
2. **Synthesize** â€” real values are swapped for convincing fakes generated by [Faker](https://faker.readthedocs.io/). Same real value always maps to the same fake within a session
3. **Forward** â€” the sanitized request is forwarded to Claude
4. **Restore** â€” synthetic values in Claude's response are swapped back to the originals before returning to the caller

Claude never sees your real PII. Your response comes back with real values intact.

## Requirements

- Python 3.10+
- ~500MB disk for the GLiNER model (downloaded on first run)

## Installation

```bash
git clone https://github.com/zakelectric/anonymyzr
cd anonymyzr

pip install -r requirements.txt

cp .env.example .env
# Edit .env and add your Anthropic API key
```

## Usage

```bash
python main.py
```

The proxy starts on `http://127.0.0.1:8080` by default.

**Point your Anthropic client at the proxy:**

```bash
# Environment variable (works for most clients)
ANTHROPIC_BASE_URL=http://localhost:8080

# Or in OpenClaw's openclaw.json:
{
  "provider": {
    "api": "anthropic-messages",
    "url": "http://localhost:8080",
    "apiKey": "sk-ant-..."
  }
}
```

## Configuration

### Environment variables (`.env`)

| Variable | Default | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | *(required)* | Your real Anthropic API key |
| `ANONYMYZR_HOST` | `127.0.0.1` | Bind address |
| `ANONYMYZR_PORT` | `8080` | Bind port |
| `ANTHROPIC_API_URL` | Anthropic default | Override Claude endpoint |
| `ANONYMYZR_LOG_FILE` | `logs/anonymyzr.log` | Log file path |
| `ANONYMYZR_LOG_LEVEL` | `INFO` | `DEBUG` / `INFO` / `WARNING` |
| `ANONYMYZR_LOG_REAL_VALUES` | `true` | Log real PII values (set `false` in production) |

### Entity types (`config/entity_types.yaml`)

Edit to add/remove what gets detected. Labels are plain English â€” GLiNER understands natural language descriptions. Adjust `threshold` (0.0â€“1.0) to tune sensitivity.

## API

The proxy exposes the standard Anthropic Messages API plus two extras:

| Endpoint | Description |
|---|---|
| `POST /v1/messages` | Standard Anthropic messages endpoint (proxied + anonymized) |
| `DELETE /v1/sessions/{session_id}` | Clear a session's PII mapping table |
| `GET /health` | Health check |

## Streaming

The proxy collects the full response before deanonymizing (required to safely swap values), then simulates an SSE stream if the client requested one. Latency impact is minimal for typical workloads. True pass-through streaming is planned for v2.
